# 五子棋游戏部署指南

## 📦 部署到 Render.com（免费）

### 前置条件
1. 注册 GitHub 账号：https://github.com
2. 注册 Render 账号：https://render.com（可用 GitHub 登录）
3. 安装 Git（如果还没安装）

---

## 第一步：上传代码到 GitHub

### 1. 初始化 Git 仓库

在项目根目录（五子棋文件夹）打开 PowerShell，执行：

```powershell
git init
git add .
git commit -m "Initial commit: 五子棋在线对战游戏"
```

### 2. 创建 GitHub 仓库

1. 访问 https://github.com/new
2. 仓库名称填写：`gobang-online`
3. 选择 **Public**（公开）
4. 不要勾选任何初始化选项
5. 点击 **Create repository**

### 3. 推送代码到 GitHub

复制 GitHub 显示的命令，或执行以下命令（替换 `YOUR_USERNAME` 为你的 GitHub 用户名）：

```powershell
git remote add origin https://github.com/YOUR_USERNAME/gobang-online.git
git branch -M main
git push -u origin main
```

---

## 第二步：部署后端到 Render

### 1. 登录 Render

访问 https://dashboard.render.com，用 GitHub 账号登录

### 2. 创建 Web Service

1. 点击 **New +** → **Web Service**
2. 连接你的 GitHub 仓库 `gobang-online`
3. 配置如下：

| 配置项 | 填写内容 |
|--------|----------|
| **Name** | `gobang-backend`（或任意名称） |
| **Region** | `Singapore`（新加坡，速度较快） |
| **Root Directory** | `backend` |
| **Environment** | `Node` |
| **Build Command** | `npm install` |
| **Start Command** | `npm start` |
| **Instance Type** | `Free` |

4. 点击 **Create Web Service**

### 3. 等待部署完成

部署需要 3-5 分钟，完成后你会看到：
- ✅ 绿色状态显示 "Live"
- 获得后端地址，格式如：`https://gobang-backend-xxxx.onrender.com`

**🚨 重要：复制这个地址（wss 协议）用于前端配置！**

例如：`https://gobang-backend-xxxx.onrender.com` → `wss://gobang-backend-xxxx.onrender.com`

---

## 第三步：配置前端环境变量

### 1. 修改前端配置

在本地编辑 `frontend/.env.production` 文件：

```env
VITE_WS_URL=wss://gobang-backend-xxxx.onrender.com
```

**注意**：
- 将 `gobang-backend-xxxx.onrender.com` 替换为你的实际后端地址
- 使用 `wss://`（安全 WebSocket 协议）

### 2. 提交修改并推送

```powershell
git add frontend/.env.production
git commit -m "Update backend URL for production"
git push
```

---

## 第四步：部署前端到 Render

### 1. 创建 Static Site

1. 在 Render Dashboard 点击 **New +** → **Static Site**
2. 选择同一个仓库 `gobang-online`
3. 配置如下：

| 配置项 | 填写内容 |
|--------|----------|
| **Name** | `gobang-frontend`（或任意名称） |
| **Region** | `Singapore` |
| **Root Directory** | `frontend` |
| **Build Command** | `npm install && npm run build` |
| **Publish Directory** | `dist` |

4. 展开 **Advanced** → **Environment Variables**，添加：

| Key | Value |
|-----|-------|
| `VITE_WS_URL` | `wss://gobang-backend-xxxx.onrender.com` |

5. 点击 **Create Static Site**

### 2. 等待部署完成

部署完成后，你会获得前端访问地址：
- 格式如：`https://gobang-frontend-xxxx.onrender.com`

---

## 第五步：测试游戏

1. 访问前端地址：`https://gobang-frontend-xxxx.onrender.com`
2. 开两个浏览器窗口，都访问这个地址
3. 两个窗口都点击"开始游戏"
4. 开始对战！

---

## 🔍 常见问题

### 1. **后端休眠问题**

Render 免费版会在 15 分钟无活动后休眠。解决方案：
- 用户访问时会自动唤醒（首次加载可能需要 30-60 秒）
- 或使用 UptimeRobot 等服务定时 ping 保持在线

### 2. **WebSocket 连接失败**

检查：
- 前端 `.env.production` 中的 URL 是否正确
- 后端是否显示 "Live" 状态
- 使用 `wss://` 而非 `ws://`

### 3. **修改代码后如何更新**

```powershell
git add .
git commit -m "描述你的修改"
git push
```

Render 会自动检测代码变化并重新部署。
