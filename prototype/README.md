# 五子棋UI原型 - 使用说明

## 📁 文件列表

```
prototype/
├── home.html       # 首页
├── matching.html   # 匹配页
├── game.html       # 游戏页
└── README.md       # 本说明文档
```

## 🚀 如何查看原型

### 方法1：直接双击打开（推荐）
1. 进入 `prototype` 文件夹
2. 双击 `home.html` 文件
3. 会在默认浏览器打开原型

### 方法2：拖拽到浏览器
1. 打开Chrome/Safari/Edge浏览器
2. 将 `home.html` 文件拖到浏览器窗口
3. 释放鼠标即可查看

## 🎮 原型功能说明

### 首页 (home.html)
- ✅ 显示游戏标题 "五子棋对战"
- ✅ 装饰性棋盘预览（带棋子）
- ✅ "开始游戏"大按钮（蓝色）
- ✅ 底部显示在线人数
- 🔗 点击"开始游戏" → 自动跳转到匹配页

### 匹配页 (matching.html)
- ✅ 标题："寻找对手中..."
- ✅ 3个圆点跳动加载动画
- ✅ 状态卡片显示排队信息
- ✅ "取消"按钮（灰色边框）
- 🔄 右上角"切换场景"按钮：可切换2种状态
  - 场景1: "正在匹配对手...当前排队: 0人"
  - 场景2: "游戏进行中,请等待...当前排队: 2人"
- ⏱️ 3秒后自动跳转到游戏页（模拟匹配成功）

### 游戏页 (game.html)
- ✅ 顶部信息栏："你是: ⚫ 黑方" + "轮到你了!"
- ✅ 15×15 Canvas棋盘（可交互）
- ✅ 示例棋局（已放置6颗棋子）
- ✅ 底部"认输"按钮（红色边框）
- 🎮 交互功能：
  - 点击空位 → 放置黑子
  - 0.5秒后 → 自动在随机位置放置白子
  - 点击已有棋子 → 无反应
- 🔄 右上角"切换状态"按钮：循环切换3种状态
  - 状态1: "轮到你了!"（蓝色）
  - 状态2: "等待对方..."（灰色）
  - 状态3: "对手断线中...剩余25秒"（橙色）
- 🏆 5秒后自动弹出游戏结束弹窗（模拟游戏结束）

### 游戏结束弹窗
- ✅ 半透明黑色遮罩
- ✅ 白色圆角卡片
- ✅ 顶部图标：🎉（胜利）
- ✅ 标题："你赢了!"（绿色）
- ✅ 副标题："五子连珠!"
- ✅ 两个按钮：
  - "再来一局"（蓝色主按钮）
  - "关闭"（灰色边框次按钮）
- 🔗 点击任意按钮 → 返回首页

## 🎨 设计规范

### 颜色
- 主色：#1890FF（高效蓝）
- 成功：#52C41A（绿色）
- 警告：#FA8C16（橙色）
- 错误：#FF4D4F（红色）
- 文字：#333（深色）/ #8C8C8C（灰色）
- 背景：渐变紫色 #667eea → #764ba2

### 尺寸
- 容器：375px × 812px（iPhone标准尺寸）
- 主按钮：240px × 56px
- 次按钮：200px × 48px
- 棋盘：330px × 330px
- 圆角：8px（按钮）/ 12px（卡片）/ 16px（弹窗）

### 字体
- 标题：32px / 28px / 24px
- 正文：16px
- 说明：14px
- 小字：12px

## 📱 适配说明

- ✅ 原型按iPhone标准尺寸设计（375px × 812px）
- ✅ 使用viewport禁止缩放，确保1:1显示
- ✅ 在电脑浏览器查看：显示为固定尺寸白色卡片
- ✅ 在手机浏览器查看：全屏显示

## 🔧 技术说明

- 纯HTML + CSS + JavaScript
- 无需Node.js或任何依赖
- Canvas绘制棋盘和棋子
- CSS动画实现加载效果
- 简单的页面跳转模拟流程

## ⚠️ 注意事项

1. **这只是静态原型**，不是真正的游戏
   - WebSocket连接是模拟的
   - 匹配和游戏逻辑是假的
   - 仅用于展示UI界面效果

2. **交互是简化的**
   - 匹配页3秒后自动跳转
   - 游戏页5秒后自动弹窗
   - 点击棋盘可以下棋（演示用）

3. **建议用途**
   - 展示给团队看界面效果
   - 给设计师做参考
   - 确认布局和配色方案
   - 作为开发时的视觉参考
